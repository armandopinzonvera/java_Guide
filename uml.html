Diagrama

Para realizar el diagrama, un alternativa es emplear la página PlantUML, en la opción de /Online Server/ nos abre una ventana para introducir código con el cual construir el diagrama
Este código va entre, una etiqueta de inicio y de final: @startuml y @enduml, consiste en primero declarar las clases, con la palabra reservada class, el nombre, llaves {}, y dentro de las llaves, declaramos los métodos() y variables.
Posteriormente indicamos la relación entre las clases con los nombres anteriormente utilizados, guiones y signo de mayorQue.

Dentro de las clases las variables se declaran como: tipo de dato y nombre, mientras los métodos con el nombre seguido de paréntesis

                            class VistaTransecto{
                                TextView longitud, latitud
                                onCreate()
                                conectarExtras()
                                putValueTextView(double latitude, double longitude)
                                stop()
                                conectarSQLite()
                                hilo()
                                onBackPressed()
                                }

Tenemos tres tipos de relaciones entre las clases:
Composición: la clase que lleva el diamante lleno, no funciona sin la clase del otro extremo, como ejemplo: computador-cpu, se indica cómo --*
Agregación: la clase que lleva el diamante vacío, tiene la del otro lado, pero no depende de ella, puede funcionar sin esa, se indica como --o
Extensión: cuando hereda, se indica como --|> 

Iniciar-->VistaTransecto
Iniciar--*Service_Location
VistaTransecto*--Service_Location
Service_Location--mapFragment
mapFragment*--VistaTransecto
VistaTransecto--ConexionSQLite
